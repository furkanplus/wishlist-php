# 🎁 Wishlist PHP — Self-Hosted Personal Wishlist

A lightweight, self-hosted, mobile-first wishlist application written in native PHP, modern CSS, and vanilla JavaScript. Avoid duplicate gifts on birthdays, holidays, or special events with a secure verification system, dynamic metadata scraping, and built-in multi-language management.

---

## ✨ Key Features

- **📱 Responsive Mobile-First Design:** A sleek, modern card grid powered by CSS Variables, Grid, and Flexbox. Looks and feels premium on any device (iOS, Android, and Desktop).
- **🔒 Anti-Duplicate Gift Verification:** When a friend marks an item as bought, they must provide a verification proof (Tracking Link or Order ID).
  - This proof is **only visible to the wishlist owner (admin)** to verify purchases.
  - Keeps the wishlist updated in real-time while preventing duplicate purchases.
- **✨ Metadata Auto-Scraper:** When adding items via the Admin Panel, inputting a product URL will automatically scrape the page title and product image (using OpenGraph, Twitter Cards, or image tag fallbacks) via cURL.
- **🗺️ Interactive Language Translation Engine:** 
  - Dynamic translation dashboard directly inside the Admin Panel.
  - Create new languages on-the-fly (e.g., `tr`, `de`, `es`), translate UI strings, and manage translation files stored in `lang/*.php` without touching code.
- **🎛️ Dynamic Admin Settings:**
  - Drag-and-drop reordering of items.
  - Set owner announcements/notes.
  - Add shipping addresses with an expiration date (for privacy, the shipping address automatically hides from the public view after a configured time).
- **🚀 Web Setup Wizard:** Interactive installation script (`install.php`) to configure the MySQL database connection, seed default settings, register the admin user, and lock down the installation automatically.

---

## 🛠️ Technology Stack

- **Backend:** Native PHP 7.4+ or 8.x
  - Requires: `PDO` (with MySQL driver), `cURL`, and `DOM`/`libxml` extensions.
- **Frontend:** Vanilla HTML5, CSS3 (Modern custom properties/variables), and Vanilla JavaScript (AJAX Fetch API, native Drag-and-Drop).
- **Database:** MySQL / MariaDB

## 🚀 Installation & Setup

### Prerequisites
- A web server running Apache/Nginx with PHP 7.4+.
- A MySQL or MariaDB database server.
- Write permissions enabled on the project folder (to write the `config.php` file).

### Step 1: Upload Files
Upload all files in this repository to your server's public directory (e.g., `/public_html` or a subfolder like `/wishlist`).

### Step 2: Run the Web Installer
1. Navigate to your website where the files were uploaded:
   ```text
   http://yourdomain.com/wishlist/install.php
   ```
2. Enter your MySQL database credentials:
   - **Host** (e.g., `127.0.0.1` or `localhost`)
   - **Port** (e.g., `3306`)
   - **Database Name** (The wizard will try to create this if it does not exist)
   - **Username & Password**
3. Create your admin login credentials.
4. Click **Install & Run Setup**.

The installer will automatically execute the schema from `schema.sql`, write `config.php`, and delete `schema.sql` for security. 

> [!NOTE]
> If your server directory is write-protected, the wizard will display the PHP configuration code block. You can manually copy it, save it as `config.php` in the root folder, and proceed.

---

## 🔒 Security Best Practices

1. **Delete/Rename `install.php`:** Once installation succeeds, `install.php` is locked. For absolute safety on production environments, manually delete `install.php` from your server directory.
2. **Access Control:** The admin area runs on `admin.php`. Access requires logging in through `login.php`, which stores user session tokens.
3. **Database Security:** `config.php` is generated with chmod `0600`/`0640` permissions where supported to prevent external reading.

## 🌐 Localization (Translations)

By default, the application is displayed in English (`en`). 
To add a new language:
1. Log in to the **Admin Dashboard**.
2. Navigate to the **Translations** tab.
3. In **Add New Language**, input a standard ISO language code (e.g., `de` for German, `fr` for French) and click **Create**.
4. Translate each interface label on-the-fly and save.
5. Visitors can now toggle between the languages using the switcher at the top of the homepage.
